II. Genèse de l'AP

Cette AP a constitué le premier projet commun de la première année de BTS SIO. Cette AP avait pour but de nous faire découvrir et utiliser Git, de nous faire expérimenter la gestion d'un projet avec la méthode agile. Enfin, elle avait surtout pour but de nous faire découvrir et utiliser le modèle MVC.


III. Environnement de travail

Voici les différents logiciels utilisés pour mener à bien cette AP :
   - Visual Studio Code
   - WampServer
   - Trello
   - Git


IV. Description de l'AP

Tout d'abord lors de cette AP, la classe était organisée en 2 groupes de 4 personnes et 1 groupe de 3 personnes. Cette AP nous faisait ainsi travailler en équipe sur une application web concernant la gestion des stages. Cette application possédait déjà le squelette d'une architecture MVC et c'était à nous de la compléter et modifier pour le bien de nos nouvelles fonctionnalités.
         Pour exécuter au mieux cette AP, la classe possédait une base de données communes, un Trello pour communiquer et nous organiser en prenant exemple sur la méthode agile. Enfin lors de l'élaboration de cette application, il nous a été imposé l'utilisation de Git. Git permet à différentes versions d'un même fichier de coexister. Les développeurs travaillant avec Git ont accès à l'historique des modifications pour l'intégralité du projet et peuvent ainsi savoir quels changements ont été faits par rapport à leur version des fichiers, qui a fait ces changements, etc. Ainsi Git nous permet de gérer les versions de l'application web et nous permet de travailler en équipe tout en séparant le travail de chacun avant son importation sur le master du projet (l'application finale). Chacun au sein de la classe avait une insertion et une consultation dans la base de données comme évolutions à effectuer. Pour ma part je devais créer une fonctionnalité permettant l'ajout d'une technologie pour un stage et une autre pour pouvoir consulter ses technologies. Pour finir, cette application web était hébergé chez un hébergeur web permetant l'accès à l'application sur le web par tous les membres de la classe.
         Malheureusement par manque de temps dû à l'appropriation d'une toute nouvelle architecture et d'1 semaine de classe en visio dû au COVID, ce projet n'a pas pu être terminé. Cependant nous retenons de cette AP : l'architecture MVC qui sera utilisé plus tard dans nos projets web ainsi que l'utilisation de Trello et Git pour nos futurs projets en groupes.
