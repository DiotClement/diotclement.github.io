II. Gènese de l'AP

Cette AP est conçue pour nous faire découvrir le fonctionnement d'une application Web en modèle MVC, la convention PSR et son utilisation. Enfin l'objectif principal de cette AP est de nous faire découvrir la méthode de diagnostic de correction d'un bug rencontré par un utilisateur. Cette application web est conçue pour le groupe pharmaceutique GSB désireux de contrôler davantage l'origine des frais de remboursement des déplacements de ses employés.

IV. Description de l'AP

Tout d'abord une fois l'application web reçu, je devais me documenter sur les conventions PSR en réalisant un document de synthèse (expliquée ce que sont les PSR, qui en sont responsables, leur utilité, ce qu'elles contiennent, donner 3 ou 4 liens intéressants). En résumé, les PSR (ou PHP Standard Recommandation) sont un ensemble de standards, des normes, spécifiques à la programmation PHP. Leur objectif est de fournir une base technique commune pour améliorer les pratiques de programmation en PHP et permettre l'interopérabilité des composants pour faire fonctionner les frameworks ensemble.
A la suite de cette documentation, je devais utiliser les conventions PSR pour toute modification ou amélioration sur l'application web.

Ensuite, un utilisateur du nom d'Agathe KYSAI m'envoya un mail me faisant part d'un bug survenu sur la fonctionnalité d'ajout d'une demande de remboursement de l'application web. Par ce bug un document fût créé ressassant la reproduction du bug, l'identification du code de l'application concernée par le bug et sa correction. Enfin un mail fût envoyé à l'utilisatrice, l'informant du bon rétablissement de la fonctionnalité.

Pour finir, un document a été créé montrant les améliorations à effectuer sur le code PHP de l'application Web, bien évidemment en prenant en compte les conventions PSR. Et dans un souci de gestion de versions de l'application web, Git et GitLab ont été utilisés.
