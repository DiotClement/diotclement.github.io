TP : Correction et évolution du site Web de l'agence Linkretz

I. Compétences mises en oeuvre

Répondre aux incidents et aux demandes d'assistance et d'évolution :
   ▸ Collecter, suivre et orienter des demandes
   ▸ Traiter des demandes concernant les applications
Développer la présence en ligne de l’organisation :
   ▸ Participer à l’évolution d’un site Web exploitant les données de l’organisation


II. Genèse du TP

Ce Tp a constitué le quatrième chapitre en bloc web de la première année de BTS SIO. Ce TP avait pour finalité la découverte et la manipulation de code HTML et CSS. Ce code était rempli de bug empêchant le bon fonctionnement d'une application web statique.


III. Environnement de travail

Voici les différents logiciels utilisés pour mener à bien ce TP :
   - Visual Studio Code
   - WampServer


IV. Description du TP

Tout d'abord, nous avons reçu une application web statique nommée Linkretz truffée de bug. Ainsi le but de ce tp était de comprendre la provenance du bug et d'effectuer sa correction. En résumé, les objectifs de ce tp était donc de savoir utiliser un fichier CSS, visualiser le code source d'une page HTML permmettant de mieux visualiser l'origine des bugs du code, savoir utiliser l'inspecteur de code, savoir modifier une règle dans une feuille de style CSS et enfin savoir modifier une page HTML.

Pour finir, voici les évolutions demandées sur le site Linkretz :
   - Affichage des caractéristiques des tour-opérateurs enregistrés dans la base de données lors du clic sur l'onglet 'TOUR-OPERATEURS';
   - Lorsque l'utilisateur sélectionne l'onglet 'CONTACT' dans le menu, il doit obtenir le nom, le prénom et le numéro de téléphone des commerciaux de Linkretz;
   - Actuellement le menu (contenu entre des balises nav) est présent dans chacun des scripts situés dans le répertoire page. Si on souhaite modifier le menu (modifier le nom d'une option, ajouter une option, supprimer une option, modifier un lien,etc.), il est nécessaire de modifier tous les scripts situés dans le répertoire page. Pour éviter ce travail fastidieux, nous allons utiliser les includes;
   - Pour la partie accessible par les employés, il faudra afficher les caractéristiques de tous les employés dans l'onglet 'CONNEXION' puis en cliquant sur le lien 'Accès au menu des salariés';
   - Pour la partie accessible par l'administrateur du site, il faudra modifier le code pour afficher les employés en vue de leur gestion (CRUD) et d'afficher les tour-opérateurs en vue de leur gestion (CRUD) dans l'onglet 'CONNEXION' puis en cliquant sur le lien 'Accès au menu de l'administrateur';
   - Modification du site de Linkretz pour offrir à un internaute la possibilité de déposer une demande d’information grâce à un formulaire dans l'onglet 'NOUS CONTACTER';
   - Ajouter une fonctionnalité permettant de créer un employé avec ses caractéristiques grâce à un formulaire dans la partie accessible par l'administrateur;
   - Mutualisation des composants graphiques d’un formulaire (ajout et modification).
